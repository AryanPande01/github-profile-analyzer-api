# 1. Unzip, install
npm install

# 2. Copy and fill in .env
cp .env.example .env

# 3. Create the DB in MySQL
CREATE DATABASE github_analyzer;

# 4. Run
npm run dev

# 5. Test it
curl -X POST http://localhost:3000/api/profiles/analyze/torvalds


--------------------------------------
What's built
4 REST endpoints:
MethodRoutePurposePOST/api/profiles/analyze/:usernameFetch from GitHub, store insightsGET/api/profilesList all stored profilesGET/api/profiles/:usernameGet one stored profileDELETE/api/profiles/:usernameRemove a profile
Extras beyond the requirements
Richer insights stored per profile — beyond just repo count and followers:

total_stars, total_forks, total_watchers across all repos
avg_stars_per_repo (influence metric)
most_starred_repo + its star count
top_languages (JSON array of top 5)
recently_active_repos (pushed to in last 6 months)
original_repos vs forked_repos
account_age_days, hireable, has_website

Production-grade extras:

Upsert logic — re-analyzing a username refreshes data, doesn't duplicate
Pagination + sorting on the list endpoint (?sortBy=total_stars&order=DESC)
Input validation with regex for valid GitHub usernames
Rate limiting (100 req/15min per IP)
GitHub 404/403 handled gracefully with helpful messages
Connection pool for MySQL (not a single connection)
Auto table creation on startup — no manual migration needed
